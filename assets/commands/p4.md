# TAGLINE

perforce client

# TLDR

**Sync workspace**

```p4 sync```

**Check out file**

```p4 edit [file]```

**Submit changes**

```p4 submit```

**View changes**

```p4 changes```

**Revert file**

```p4 revert [file]```

**Show file status**

```p4 status```

# SYNOPSIS

**p4** [_command_] [_options_]

# PARAMETERS

**sync**
> Sync workspace.

**edit** _FILE_
> Check out file.

**submit**
> Submit changelist.

**changes**
> List changelists.

**revert** _FILE_
> Revert changes.

**status**
> Show file status.

**--help**
> Display help information.

# DESCRIPTION

**p4** is the Perforce client. Manages version control with central server.

The tool provides enterprise version control. Handles large binary files.

# CAVEATS

Requires Perforce server. Commercial product. Enterprise focused.

# HISTORY

Perforce was created in **1995** as an enterprise **version control system**.
