# TLDR

**Open binary for analysis**

```r2 [binary]```

**Open in write mode**

```r2 -w [binary]```

**Open with analysis**

```r2 -A [binary]```

**Debug mode**

```r2 -d [binary]```

**Open remote file**

```r2 [gdb://host:port]```

# SYNOPSIS

**r2** [_options_] [_file_]

# PARAMETERS

_FILE_
> Binary file.

**-A**
> Run analysis.

**-w**
> Write mode.

**-d**
> Debug mode.

**-a** _ARCH_
> Architecture.

**-b** _BITS_
> Bits (32/64).

**-c** _CMD_
> Run command.

# DESCRIPTION

**r2** is radare2 reverse engineering framework. Binary analysis tool.

The tool disassembles and analyzes binaries. Advanced reverse engineering.

r2 analyzes binaries.

# CAVEATS

Steep learning curve. Powerful but complex.

# HISTORY

radare2 was created as a **reverse engineering** framework.
